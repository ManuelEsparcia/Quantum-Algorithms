Grover's Search

A needle in a haystack, found in √N tries instead of N — and the geometry that explains it.

A needle in a haystack.

A new kind of box, and a very different kind of problem. This one takes an n-bit input and answers a single yes/no question: "is x the one I'm looking for?" Among the N = 2ⁿ possible inputs, exactly one — the marked item — returns 1. Everything else returns 0. Your job: find the marked item.

The oracle

All zeros, except one.

A checker. For the marked item m it returns 1. For every other input, 0.

What you can't do

No structure.

Unlike every previous box in this series, there's no algebraic shortcut. The marked item has no periodic structure, no linear pattern — it's just somewhere. The haystack is truly unstructured.

The question is simple. Given this kind of box, how many queries do you need to find the marked item?

Search it classically.

Below is a grid of 16 boxes. Exactly one of them is marked — the rest are empty. Click boxes one at a time to check them. On average you'll need N/2 = 8 checks to find the marked one. In the worst case, all 16.

How unstructured search works

With no clues about where the marked item is, every box you check is as likely as any other to hold the answer. The best classical strategy is simply to check them one by one in some order. After k checks, the probability of having found the marked item is k/N. There's no trick that does better in general.

For N = 16 that's up to 16 queries. For N = 10⁶, up to a million. A quantum computer, as you'll see, does it in √N — so 4 queries for N = 16, and a thousand for N = 10⁶.

Classical search scales linearly: to find one marked item in N, you'll typically check N/2 and in the worst case N. Grover's algorithm takes about √N queries. That's the quadratic gap we're going to build in the next sections.

Quadratic, not exponential.

Classical unstructured search over N items costs N/2 queries on average, N in the worst case. Grover's algorithm needs about √N. This is a quadratic speedup — less dramatic than Simon's exponential gap, but it applies to any search problem you can pose as "is this the answer?" That generality is why Grover matters.

Rotations, not cancellations.

Grover's algorithm works differently from the three that came before. DJ, BV, and Simon rely on phase kickback followed by a Hadamard collision that cancels everything except the answer. Grover uses a subtler mechanism: each iteration rotates the quantum state a tiny angle closer to the marked item. Do the rotations the right number of times and you land exactly on the answer.

Start, as usual, with a uniform superposition of all N inputs. Now split that state into two parts: the amplitude on the marked item, and the amplitude on everything else. Those two pieces span a 2D plane. Every step of Grover's algorithm is just a rotation inside that plane.

The register starts in the state |0…0⟩. One amplitude is 1, the rest are 0.

At this point every one of the N = 8 amplitudes is 1/√8 ≈ 0.354. The marked item has the same amplitude as every other item — we haven't done anything yet. What Grover does next is ingenious: an oracle flips the sign of the marked amplitude, then a diffusion step reflects everything around the average. The geometric effect: the state rotates a little toward the marked item. Each iteration rotates again. After about (π/4)√N rotations, the state points almost exactly at the marked item — and a measurement finds it.

The optimal number of iterations is about (π/4)√N. For N = 8 that's about 2 — and indeed, after 2 Grover iterations the probability of measuring the marked item is 94.5%. A third iteration would actually reduce it: keep rotating and you overshoot the target. Grover's isn't a "more is better" algorithm.

Amplitude distribution —

—

One counterintuitive feature of Grover's: the probability oscillates. After the optimal k* iterations it's maximal; one more iteration and it decreases. The state is still rotating in the plane — we've just gone past the marked axis. Set k well beyond the optimum and you can drive the probability back near zero.

Square roots everywhere.

Grover's algorithm is the most widely applicable result in this series. Any problem where you can check a candidate answer in polynomial time — SAT, graph coloring, subset-sum, password brute force — inherits a quadratic speedup from Grover. It's not the earth-shattering advantage of Shor, but it's a universal one.

It's also provably optimal: no quantum algorithm can search an unstructured haystack faster than √N. Grover is the speed limit. That makes it the natural cryptographic threat model: doubling a symmetric key length is a reasonable defense against a quantum-equipped adversary, because Grover cuts effective key strength in half, not to zero.
