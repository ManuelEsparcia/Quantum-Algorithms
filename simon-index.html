A function that echoes.

A new kind of box. This one takes n bits in and returns n bits out. The only promise: the function f is two-to-one, and the two inputs that map to each output always differ by the same hidden bit-string s. Formally, f(x) = f(x ⊕ s) for every x, and that's the only collision.

What is s

The hidden period.

A fixed, unknown, non-zero bit-string of length n. Here's an example.

What you'll see

The pairing.

Every output value is produced by exactly two inputs — a pair (x, x⊕s). Four outputs for n=3, each one hit twice.

The question is simple. Given this kind of box, can you find the period s? And if so — how many queries do you need?

Hunt for a collision.

Below is a box with a 4-bit hidden period s. If you ever query two different inputs x₁ and x₂ with f(x₁) = f(x₂), you've found a collision — and the period is just their XOR: s = x₁ ⊕ x₂. Your job is to stumble onto one.

How the box works

The function is 2-to-1 with hidden period s. Out of 16 possible inputs, only 8 distinct output values exist — each output value appears exactly twice, once for some x and once for x ⊕ s.

Classical strategy: query random inputs until two of them return the same output. The more queries you make, the more likely a collision becomes — this is the birthday paradox. For n = 4 you'll typically find one in 4–6 queries. For n = 100: around 2⁵⁰ queries. That's the problem.

    A handful of queries was enough at n = 4. But this approach is fundamentally limited — the expected number of queries grows as √2ⁿ. Double the bits and you quadruple the work. The quantum algorithm, as you'll see, only needs about n.

    The birthday cost.

    For an n-bit period, the best any classical algorithm can do is about √2ⁿ queries on average — the birthday paradox in action. Simon's quantum algorithm needs only about n queries. For the first time in this series, the gap is a provable exponential advantage over every classical algorithm — deterministic or randomized.

    Superposition, but now with partners.

    Simon's algorithm uses the same building blocks as Deutsch–Jozsa and Bernstein–Vazirani — uniform superposition, oracle call, interference — but with one new twist: the output register is entangled with the input register. That entanglement conspires with the final Hadamards to reveal something unexpected: not the period itself, but a random string perpendicular to it.

    To refresh the picture: one Hadamard gate on each of three qubits turns |000⟩ into a uniform superposition of all 8 three-bit strings. Every possible input, at once — ready to be fed to the oracle.

    The register starts in the state |000⟩. One amplitude is 1, the rest are 0.

    What happens next is the crux. After the oracle marks each input with its f-value, the second layer of Hadamards creates a strange interference pattern: the only input states with non-zero amplitude are those y for which s · y = 0 (mod 2). Every measurement gives one such y. Collect n − 1 independent ones and a dash of linear algebra recovers s.

    Each run gives one y satisfying s · y = 0. One run isn't enough to recover s — each y is a linear constraint, and you need n − 1 independent ones to pin the period down. That's where the next section comes in.

    Putting it together.

    One run of the quantum algorithm gives one constraint s · y = 0. That's a single linear equation in n unknowns over the field of two elements. To pin down a non-zero s you need n − 1 independent equations. Run the algorithm here, collect y's, watch the system of equations build up, and recover the period.

    Some shots are useless — a y that's already in the span of previous ones adds no new information, and the zero string y = 00…0 is trivially orthogonal to everything. On average you'll need about n + 3 shots to gather n − 1 independent ones. Still exponentially better than classical.

    The template for Shor.

    Simon's algorithm is the first in this series that gives an exponential oracle speedup over any classical algorithm, deterministic or randomized. But it's more than a curiosity: it is the conceptual ancestor of Peter Shor's factoring algorithm. Replace "hidden period of a function on bit-strings" with "hidden period of modular exponentiation," replace Hadamards with the quantum Fourier transform, and you have Shor.

    The lesson that carries forward: when a problem has a hidden algebraic structure — a period, a symmetry, a subgroup — quantum computers can expose it in polynomial time, even when classical computers cannot. This is the real engine of quantum advantage.
