Somewhere, a hidden string.

A new box. Inside it lives a fixed bit-string s of length n — some specific sequence of zeros and ones, chosen once and then kept out of sight. You cannot read it directly. The only thing the box will do for you is this: give it any input x, and it returns the bit s · x (mod 2) — the parity of the bits of x in the positions where s has a one.

What you can ask

The oracle.

For any x, the box returns s · x — the XOR of the bits of x where s has a one.

The question is simple. Given this kind of box, can you recover s? And if so — how many queries will you need?

Recover the secret, classically.

Below is a box hiding a 4-bit secret s. Give it any 4-bit input x and it returns a single bit: the XOR of the bits of x in the positions where s has a one. Your job is to work out what s is.

How the box works

The box computes s · x (mod 2). Suppose for a moment that s = 1011. Then:

  • s · 0001 = (1·0) ⊕ (0·0) ⊕ (1·0) ⊕ (1·1) = 1  →  you just learned s0 = 1
  • s · 0010 = (1·0) ⊕ (0·0) ⊕ (1·1) ⊕ (1·0) = 1  →  you learned s1 = 1
  • s · 1111 = 1 ⊕ 0 ⊕ 1 ⊕ 1 = 1  →  you learned the total parity of s, but no single bit

That's the trick: a query whose input has a single 1-bit reads one bit of s directly. The four quick-query buttons below do exactly that. Custom queries give you XOR combinations — useful, but less direct.

    The linear cost.

    An n-bit secret carries n bits of information. Every classical query returns a single bit. So no deterministic strategy can ever do better than n queries. Quantum: always one, regardless of n.

    The same trick, a cleaner payoff.

    If you already read the Deutsch–Jozsa entry, the quantum recipe will look familiar: a uniform superposition of all inputs, a single oracle call, then an interference step that concentrates the amplitude on the answer. Here the payoff is sharper — the final measurement returns the secret s directly.

    To refresh the picture: a qubit does not have to sit in the state 0 or 1. It can be a blend of both, described by amplitudes. A single Hadamard gate on each of three qubits turns |000⟩ into a perfectly uniform superposition of all 8 three-bit strings. Every possible input, at once.

    The register starts in the state |000⟩. One amplitude is 1, the rest are 0.

    Superposition puts all inputs on the table. The oracle marks them with phases. The second layer of Hadamards collides those phases so that everything cancels except one amplitude — and that amplitude sits exactly on |s⟩.

    The cleanest speedup.

    Bernstein–Vazirani feels, at first, less impressive than Deutsch–Jozsa — the gap is n-to-one, not exponential. Look closer and the roles reverse. Deutsch–Jozsa is contrived: nobody ever hands you a function promised to be constant or balanced. Bernstein–Vazirani is a real pattern. A hidden linear structure revealed in a single query. This is the template for real quantum advantages.

    Think of it this way: BV is how DJ wanted to grow up. It has the same elegant skeleton — superposition, phase kickback, interference — but a cleaner question and a cleaner answer. From here, the natural next step is Simon's algorithm, which trades linear structure for hidden period structure and gives us the first exponential speedup over any probabilistic classical algorithm.
