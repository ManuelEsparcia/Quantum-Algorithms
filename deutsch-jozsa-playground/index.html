A promise about a box.

Somewhere in front of you there is a black box. Inside it lives a function f that takes n bits in, and spits one bit out. You do not know what the function is. The only thing you are promised is this:

Option A

Constant.

The function always returns the same answer, for every input. All zeros, or all ones.

Option B

Balanced.

The function returns 0 for exactly half of the inputs, and 1 for the other half.

The question is simple. Given this box, can you tell whether it is constant or balanced? And if so — how many times do you need to look inside?

If you were unlucky, you queried four inputs and they all came back the same. You probably thought "must be constant" — but it could still be a balanced function that happens to return 0 on all four of them. To be sure, you need to see one more input.

The scaling curse.

The pattern is merciless. For an n-bit function you may need 2n−1 + 1 queries in the worst case to be fully certain. That is exponential in the number of input bits.

What if you could ask about all inputs at once?

Classical computers look at inputs one at a time. Quantum computers can prepare a superposition of every possible input — and feed them all into the oracle in a single call.

A quantum bit (a qubit) doesn't have to be in the state 0 or 1. It can be in a blend of both, described by numbers called amplitudes. Below: press the button to apply a Hadamard gate and see a single qubit split into a uniform superposition of the 8 possible 3-bit inputs. All of them, at the same time.

The register starts in the state |000⟩. One amplitude is 1, the rest are 0.

But superposition alone is not enough. If we measured now, we would get one random input out of eight — same information as one classical query. The trick is next: use interference to make the wrong answers cancel out, and the right one add up.

The punchline: if the function is constant, all amplitudes pile up on |000⟩ — so the measurement outputs 000 with certainty. If the function is balanced, the amplitude on |000⟩ is exactly zero — so the measurement never outputs 000. One shot is enough.

Why this matters.

The Deutsch–Jozsa problem is, admittedly, contrived. Nobody is ever going to hand you a function and promise that it is constant or balanced. But the algorithm matters for a different reason: it is the first time we proved that a quantum computer can beat every possible classical deterministic algorithm, and by an exponential margin.

More importantly, Deutsch–Jozsa is the template. Every subsequent quantum algorithm in this series uses the same skeleton — prepare a superposition, let the oracle imprint a phase, then use interference to reveal a hidden structure. Master this circuit and you have the foundation for all the rest.
